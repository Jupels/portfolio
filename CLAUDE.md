# alexlgncom — Правила проекта

## Язык
- Комментарии в коде: русский
- Документация: русский
- Общение: русский

## Стек
- **Frontend:** React 19 + TypeScript + Next.js 16 + Tailwind CSS 4
- **State:** Zustand (persist в localStorage)
- **Backend:** Next.js API Routes + OpenAI API
- **Валидация:** Zod
- **БД (планируется):** Supabase (PostgreSQL + Auth + Storage)

## Перед началом работы
- Прочитать `docs/MAP.md` — обзор проекта и карта модулей
- Прочитать `docs/ROADMAP.md` — план развития
- Проверить `implementationPlan.md` — незавершённые задачи

## Логирование
- Каждое изменение → новая строка в `log.md`
- Формат: `[дата] [session-id] — описание изменения`
- Новые запросы отделять пустой строкой

## Документация
- При изменении компонента → обновить `docs/`
- Нет документации → создать задачу, согласовать с пользователем

## План работы (implementationPlan.md)
- Новая задача → записать запрос + план решения
- Session ID: `date +%s`
- Вести TODO, отмечать выполнение
- Завершение → пометить план + запись в log.md

## Проверка кода
```bash
npm run lint    # проверка синтаксиса
npm run build   # сборка
npm run dev     # dev-сервер (только для UI)
```
Тесты — запрашивать подтверждение перед запуском.

## Принципы
- SOLID, KISS, DRY
- Безопасность и производительность
- Минимально необходимое, без переусложнения
